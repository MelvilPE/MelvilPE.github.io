DAVA Revanced is a reworked and enhanced continuation of DAVA.Framework.

üìò ResourceEditor

ResourceEditor is the main tool for modders to create and edit Blitz Tank Models. It allows working with the 3D scene and supports the latest model versions.

The original DAVA.Framework supported up to scene file version 24 (Around 2018‚Äì2019, World Of Tanks Blitz 5.9). Development of the original framework is now hidden from the public.

Our modifications enable loading and saving scenes in version 48, the latest released by Wargaming and Lesta Studio.

New features include:

  • Extended Features:
    • YAML editing of unregistered components
    • Global scene information management
  • Material Editor Improvements:
    • New flags and new presets properties
    • Support for custom cull modes (FRONT, BACK, DEFAULT, NONE)
  • Extended Compatibility:
    • Load and save new tank models compatible with the latest scene version

üìó Unregistered Components

Unregistered Components are components that are loaded but not supported by the current version of the framework.
This new feature allows load and edit components content in a YAML-based approach.

üìó Marked Unregistered

Marked Unregistered is a new option has been added to the general ResourceEditor settings.
This new feature allows to mark 3D components from the engine as unregistered.
This way, if new settings are introduced or the component undergoes major changes, you can disable it and switch to a YAML-based approach.

keyedArchive:
    list:
        variantVector:
        -   string: "SlotComponent"

üìó Scene Infos

Scene Infos panel has been updated with new YAML-based features.
- Scene Components and Scene Component Sets are components that apply to the entire scene.
- Scene Render Config replaces the old Global Material.
- Particle Emitter Nodes replace old YAML files system, with new attributes and variables.

Note: Implementing these new features in a graphical interface would be complex.
And updates may differ between Wargaming and Lesta.
For now, keeping them in YAML is more practical, especially since this kind of content could change often.

üìò FormatEditorCLI

FormatEditorCLI is a CLI tool designed to convert files between specific formats and YAML, and vice versa, allowing modders to edit file contents textually.

Available methods/options:

  • Method 0: Convert KeyedArchive files to YAML
  • Method 1: Convert YAML files to KeyedArchive
  • Method 2: Convert Scene files to YAML
  • Method 3: Convert YAML files back to Scene files
Command example: FormatEditorCLI -i E:\OS\input.sc2 -o E:\OS\output.yaml -m 2
Note: For scene files, only versions greater or equal to 30 are supported by this tool.

üìò ResourceDLC

ResourceDLC is a lightweight command-line tool for working with SmartDLC package files in DVPK and DVPD + DVPM formats.

Available methods/options:

  • Method 0: Unpack DVPK / DVPM+DVPD to output directory
Command example: ResourceDLC -i E:\OS\input.dvpm -o E:\OS\Output\ -m 0
